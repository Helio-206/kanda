# Web Scope

## What the Web is for

- Landing Page
- Dashboard das Entidades
- Dashboard Administrativo
- Insights
- Mapa Administrativo

The Web is the institutional and operational face of KANDA. It is built for entities, oversight, and territorial reading.

## What the Mobile app is for

The mobile app is the main citizen experience.

It will handle:

- reporting;
- exploration;
- tracking;
- confirmations;
- credibility.

## Legacy Web pages

The following pages still exist in the Web codebase, but they are legacy/prototype surfaces and are no longer part of the main navigation:

- `/reportar`
- `/acompanhar`
- `/ocorrencia/:codigo`
- `/legacy/reportar`
- `/legacy/acompanhar`
- `/legacy/ocorrencia/:codigo`

These pages stay temporarily to avoid breaking the current prototype while the mobile experience takes over the citizen flow.

## Official Web routes for the MVP

- `/`
- `/dashboard-entidades`
- `/dashboard-administrativo`
- `/insights`
- `/mapa-administrativo`

## Notes

- The Landing Page should stay visually strong, but the CTA language should point to the platform and the panel, not to citizen reporting.
- Citizen reporting and tracking are future mobile-first flows.
- The Web should read like a platform for institutions, not a consumer app.
