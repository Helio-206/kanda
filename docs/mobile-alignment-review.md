# Mobile Alignment Review

This review reflects the mobile app after it was folded into the same repository as the Web app.

## What changed

- Home now speaks the language of the KANDA platform, not just a local reporting app.
- Exploration became a visible first-class action from the home experience.
- The report flow now ends with a stronger success moment.
- Detail screens now surface score, entity, and community impact more clearly.
- Profile now includes a minimal but visible credibility story.
- Map and badges were nudged closer to the Web identity.
- Tabs and copy were adjusted to feel more institutional and less ad hoc.

## Problems found

- The mobile still looked too much like a self-contained prototype.
- Exploration was not obvious enough from the first screen.
- Credibility existed only in architecture, not in the UI.
- The submit flow ended too quietly.
- The detail page read like a local record, not an active case.

## Problems solved

- Exploration is now clearly presented as a central user action.
- Credibility is visible in the Home and Profile surfaces.
- Submission now ends on an official registration moment with a real follow-up path.
- The detail view now reads more like an ongoing case.
- The overall tone is closer to the institutional KANDA Web experience.

It reads less like an isolated denunciation tool and more like the citizen layer of a serious platform.

It is not finished, but it is now defensible in a hackathon demo.
